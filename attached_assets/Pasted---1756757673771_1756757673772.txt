ТОЧНО ЧТО НУЖНО СДЕЛАТЬ:

1. В админ‑панели при создании тура все эти составляющие должны выводиться списком.  
   - Каждая строка = чекбокс + поле для цены в TJS (как сейчас это сделано с типами номеров в гостиницах).  
   - Администратор может отметить, какие составляющие включить в тур, и указать цену для каждой.  

2. Система должна автоматически суммировать цены отмеченных составляющих и показывать итоговую цену тура внизу формы.  
   - Итог показывать числом: «Цена за тур: XXX TJS».  

3. Данные при сохранении тура:  
   - Сохранять список выбранных составляющих и их цены (например JSON).  
   - Сохранять итоговую цену тура (totalPriceTJS).  

4. При выборе города для тура:  
   - Должна происходить фильтрация отелей.  
   - В списке выбора отелей должны показываться только те отели, которые находятся в этом же городе.  

5. В админ‑панели есть раздел «Калькулятор цен» (список составляющих).  
   - Нужно сделать рабочую логику: в этом модуле админ один раз указывает цены для всех составляющих (строго из списка выше).  
   - Эти базовые цены автоматически подставляются в форму создания тура в соответствующие поля.  
   - Админ при создании тура может корректировать их вручную, но по умолчанию они должны браться из калькулятора.  

6. На фронтенде (для пользователей):  
   - Цена тура должна формироваться на основе указанных составляющих (то есть не хранить в базе «фейковую фиктивную цену», а именно использовать сохранённый расчёт).  
   - Для отображения на сайте брать totalPriceTJS.  
   - Для переключателя валют (TJS/USD/EUR) продолжать использовать модуль конвертации, но базовой валютой остаётся TJS.  

---

❗ ИТОГ:
- При создании тура цена задаётся не вручную, а формируется из фиксированного списка составляющих.  
- Каждая составляющая = чекбокс + поле для цены (в TJS).  
- Итоговая цена автоматически считается и сохраняется.  
- Фильтрация отелей по городу должна работать.  
- Калькулятор цен в админке должен быть рабочим источником, откуда подтягиваются базовые цены для всех составляющих.
