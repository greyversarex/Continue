ТРЕБОВАНИЕ I: ПЕРЕХОД НА ДИНАМИЧЕСКУЮ СТРУКТУРУ
Централизация Шапки и Подвала:
Эталон: Используй HTML-код для хедера и футера исключительно из файла frontend/index.html.
Создай _header.html и _footer.html в папке frontend/.
Напиши универсальный JS-скрипт (layout-loader.js), который будет асинхронно загружать и вставлять _header.html в начало <body> и _footer.html в конец <body>.
Примени этот механизм ко всем страницам, удалив из них дублирующийся код хедера и футера.
Рефакторинг JavaScript:
Для всех страниц, содержащих значительную JS-логику в тегах <script>, вынеси этот код в отдельные, хорошо именованные .js файлы в папку frontend/public/js/.
ТРЕБОВАНИЕ II: РЕАЛИЗАЦИЯ ПОЛНОЦЕННОЙ ТРЕХЪЯЗЫЧНОЙ ПОДДЕРЖКИ
Настройка Языков:
Система должна поддерживать три языка: Английский (en), Русский (ru), Таджикский (tj).
Английский (en) должен быть языком по умолчанию при первой загрузке сайта.
Переключатель языков на сайте должен содержать только эти три опции.
Админ-панель:
Во всех формах создания и редактирования сущностей (Туры, Отели, Новости и т.д.) должны быть отдельные, четко подписанные поля для ввода контента на трех языках (например, "Название (EN)", "Название (RU)", "Название (TJ)").
Бэкенд должен корректно сохранять эти данные в едином JSON-объекте ({"en": "...", "ru": "...", "tj": "..."}).
Фронтенд:
Статический интерфейс: Проведи полную ревизию и заверши на 100% работу по добавлению data-translate атрибутов и наполнению словаря i18n.js для всех элементов на всех страницах. Словарь должен содержать переводы на все три языка.
Динамический контент: JavaScript-код, отображающий данные из базы (названия туров, описания отелей), должен корректно выбирать и показывать текст в зависимости от выбранного языка (en, ru или tj).
